<h1 align="center">Resume Website – Full Stack Portfolio</h1>

> A personal resume and portfolio website built to present my academic background, technical skills, and projects in a clear, professional, and modern way.

## Overview

This project is a **full stack resume and portfolio website** designed to showcase my profile as a **Computer Science student and software developer in training.**

The goal is to provide recruiters and collaborators with a clear overview of my skills, education, and projects, while also demonstrating practical knowledge of **frontend–backend integration**, clean code organization, and user-focused design.

The website follows a **dark, cyber / code-inspired aesthetic**, with subtle references to terminal interfaces and minimal visual effects.

## Key Features

- Responsive resume and portfolio layout

- Multilingual support:
  - English (default)
  - Portuguese
  - Spanish

- Organized sections:
  - About
  - Skills
  - Academic & Technical Background
  - Projects
  - Contact

- Functional contact form with backend integration
- Server-side data persistence for messages
- Clean separation between frontend and backend

## Tech Stack

### Frontend
- HTML5
- CSS3
- JavaScript (Vanilla)

### Backend
- Node.js
- Express

### Data & Communication
- REST endpoint for contact form
- JSON file storage

### Tools
- Git & GitHub
- Visual Studio Code

## Design & Visual Identity

- **Theme:** Dark

- **Style:** Cyber / code / minimal
  
- **Visual language:**
  - Clean typography
  - Card-based layout
  - Subtle hover effects
  - Terminal-inspired structure

Supported color references used during design discussions:
  - Primary dark background: `#000000`
  - Light text contrast: `#ffffff`
  - Accent highlights (planned): `#0969DA`

## Getting Started

### Prerequisites
- Node.js (v18 or higher recommended)
- npm

### Backend Setup

```bash
cd server
npm install
npm start
```

The backend will run at:

```bash
http://localhost:3000
```

### Frontend

Open `index.html` directly in your browser or serve it using a local static server.
